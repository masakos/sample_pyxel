# 20分ワークショップ

##  全体の流れ


|   時間 |   内容
|------|------
|0〜5分   | ゲームの仕組みを説明
| 5〜17分  | 改造タイム
| 17〜20分 | 改造を発表・遊ぶ

  ---
  説明フェーズ（5分）でのポイント

  ゲームループの概念を1枚の図で見せる

  毎秒30回くり返す
    ┌─────────────────────┐
    │  update() 状態を更新  │  ← キー入力・移動・当たり判定
    │  draw()   画面を描く  │  ← 毎回まっさら→描き直し
    └─────────────────────┘

  「アニメのパラパラ漫画と同じ！速くめくると動いて見える」

  変数＝ゲームのパラメータという見せ方

  MOVE_SPEED = 2   # ← ここを変えるだけでキャラの速さが変わる
  JUMP_POWER = -5  # ← マイナスは「上方向」という意味
  GRAVITY    = 0.3 # ← これが大きいとすぐ落ちる

  「数字を変えるだけでゲームが変わる」という体験が最初の感動ポイントです。

-  コードを変更したら、一度ゲームを終了して再起動する必要があります。

  ---
  改造タイム（12分）の設計

  3段階のお題カードを用意（早い子は上、不安な子は下から）

  レベル1「数字をいじる」（2〜3分）

  ファイル冒頭の定数を変えて遊ぶだけ。成功体験をまず作る。

  # ★ ここを自由に変えてみよう！
  MOVE_SPEED = 2    # 大きくすると速くなる（最大10くらい）
  JUMP_POWER = -5   # マイナスを大きくすると高く飛ぶ（-10くらいまで）
  GRAVITY    = 0.3  # 大きくするとすぐ落ちる
  ENEMY_INTERVAL = 60  # 小さくすると敵が増える（最小20くらい）

  レベル2「スコアを追加する」（5分）

  コードの指定箇所にコピペで動く。達成感が出る。

  # update() の中に追加
  self.score += 1

  # draw() の中に追加
  pyxel.text(5, 5, f"SCORE: {self.score}", 7)

  レベル3「ゲームを自分流にする」（残り時間）

  - 敵の色・形を変える
  - ゲームオーバー時にメッセージを出す
  - 背景色を変える（pyxel.cls(色番号)）
