

## ゲームが動く仕組み

![ゲームが動く仕組み](./image/game-system-overview.png)

- ゲームは アニメのパラパラ漫画と同じです！
  - ゲームは1秒間に30回、以下のサイクルを猛スピードで繰り返しています。
  - 「状態を更新（update）」：頭脳のフェーズ
    - キー入力の受け付け、キャラクターの移動、当たり判定の計算を行います。
  - 「画面を描く（draw）」：見た目のフェーズ
    - 前の画面を一度真っさらにしてから、新しい位置に描き直します。
    - playerの位置を毎回計算して、画面上のどこに表示するかを決定して、表示している
- キャラクターが何かにあたったかどうかを判定するロジックは、キャラクターと対象物で重なっているところがあるかで判定します。


## 遊び方
-  左・右矢印キー: 左右に移動
-  スペースキー： ジャンプ
　
## 改造タイム
- プログラムの中にある数字を変えるだけで、ゲームの世界の物理法則を書き換えることができます！
- コードを変更してみて、一度ゲームを終了して再起動て動作を確認してみましょう。

プログラミングは、まずは自分のくわえた変更によって、どのように動きが変わるかを確認する作業を繰り返すのが上達の一歩です!!

### 【レベル1】ゲームバランスを支配する（難易度：★☆☆）
ファイル冒頭にある「定数」の数字を書き換えて動作を確認してみましょう
- MOVE_SPEED: キャラクターの移動速度
- JUMP_POWER: ジャンプの高さ
- GRAVITY: 重力の強さ
- ENEMY_SPEED: 敵の移動速さ
- ENEMY_INTERVAL: 敵が出る間隔


-----
### 横移動
- MOVE_SPEED 
  - 右キー押す → x座標を少し足す
      - フレームごとに → 2ピクセルずつ右にズレる

```
プレイヤーは常に右へ動いている（world座標は増え続ける）
一定位置を超えるとカメラも同じだけ動き、位置を打ち消す
その結果、プレイヤーは中央に見え、背景が左に動いて見える
```

- ゲーム画面であれば、速度は1フレームで1ピクセル動く場合、1ピクセル/フレームです。

### ジャンプ
- JUMP_POWER 
  - スペースキー入力時に縦速度（player_vy）をこの値（負数）に設定する。上向きの初速が強くなり、高く跳べる。
- GRAVITY 
毎フレームplayer_vy += GRAVITYで下向きに加速。値が大きいほど落下が速く、重い感じになる。

- ジャンプは「速度」で表現している
    - 最初：上向きの勢い（vyがマイナス）
    - 重力でだんだん弱くなる
    - 0になる → 頂点
    - プラスになる → 落下

→　だから「山なりのカーブ（放物線）」になる


```
なぜ、現実世界の地球の重力加速度 9.8 m/s² だとうまく動かないのか?

現実の「9.8」は → 1秒あたりの加速度（m/s²）

一方、Pyxel では「1フレームごとに +9.8 速度が増える」ため、一瞬で落下が爆速になる。結果ジャンプがほぼ機能しない。

- 座標は「ピクセル」
- 更新は「1フレームごと（通常 30fps）」
- 単位はすべて フレーム単位の擬似物理
```


### 【レベル2】「スコア」を表示する（難易度：★★☆）

- update() の中に self.score += 1 の処理で生き残るほどスコアが増えるようになっています
- draw() の中に pyxel.text(...) を追加して、画面にスコアを表示させてみましょう
    -  現在この処理がコメントアウトされていて処理が実行されないようになっています。該当部分を探して、コメントをはずしましょう。

```python
# update() 
self.score += 1

# draw() の中に追加
pyxel.text(5, 5, f"SCORE: {self.score}", 7)
```

- 第1引数 5：テキストの x 座標（画面左からの距離）
- 第2引数 5：テキストの y 座標（画面上からの距離）
- 第3引数 f"SCORE: {self.score}"：表示するテキスト（スコア値を埋め込み）
- 第4引数 7：テキストの色番号（Pyxelのパレット色。例：7は白）
